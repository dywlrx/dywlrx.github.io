在windows7系统中使用Gridea遇到的问题和解决办法

2019-08-08 1 min read

我是在少数派看到关于Gridea的介绍的，觉得很是方便，很有兴趣尝试。

本人并不熟悉git和github的操作，我参照介绍教程一步步下载安装配置，但在使用过程中遇到了一些麻烦。

一直提示远程连接失败

按照教程，在github.com里都设置好了，但是到了Gridea配置界面就是连接不上。

从Gridea官网查询得知，可在Gridea程序的“开发者工具”中获取错误信息。我的错误信息如下：

message: "fatal: No such remote 'origin'↵"

在CSDN网站找到解决的办法：链接

提示出错信息：fatal: remote origin already exists.
解决办法如下：
1、先输入$ git remote rm origin
2、再输入$ git remote add origin git@github.com:djqiang/gitdemo.git 就不会报错了！
3、如果输入$ git remote rm origin 还是报错的话，error: Could not remove config section 'remote.origin'. 我们需要修改gitconfig文件的内容
4、找到你的github的安装路径，我的是C:\Users\ASUS\AppData\Local\GitHub\PortableGit_ca477551eeb4aea0e4ae9fcd3358bd96720bb5c8\etc
5、找到一个名为gitconfig的文件，打开它把里面的[remote "origin"]那一行删掉就好了！

但是，直接按照上面的操作还是不行的，经一番研究之后，正确做法如下：

  1. 打开Git Bash，进行初始化设置，使用git config user.name "name"和 git config user.mail "e-mail"
  2. cd到..\Gridea\output，在此目录下面进行操作git init
  3. 修复错误，先输入git remote rm origin，再输入git remote add origin git@github.com:name/name.github.io.git

至此，回到Gridea程序，应该可以连接成功，同步完成后，就可以浏览你的网站了。
